Objects... So What?

  • No such thing as primitive types (int, float, etc).
  • Instead everything is an object: class Fixnum, class Float.
  • EVERYTHING has an interface for method invocations.

Dynamic Open Classes

At any given time you can open up ANY class and modify or add funcionality.


          class Array
            def sum
              inject {|sum, x| sum + x}
            end
          end

          [1,2,3].sum # 6

Modules


          class Person
            attr_access :name

            include palindrome
          end

          class Car
            attr_accessor :name

            include palindrome
          end

          module Palindrome
            def palindrome?
              @name == @name.reverse
            end
          end

Modules


          austin = Person.new
          austin.name = "Austin"
          austin.palindrome? # false
          austin.name = 'foooof'
          austin.palindrome? # true

Modules


          new_car = Car.new

          new_car.palindrome? # false

Test/Unit

  • Test unit is a standard library in Ruby!
  • Most Ruby projects subscribe to the TDD paradigm

Test/Unit


          class Person
            attr_accessor :first_name, :last_name

            def full_name
              "#{@first_name} #{@last_name}"
            end
          end

          person = Person.new
          person.first_name = "Austin"
          person.last_name = "Fonacier"
          person.full_name # "Austin Fonacer"

Test/Unit


          require 'test/unit'
          require 'person'

          class PersonTests < Test::Unit::TestCase
            def test_full_name_is_first_and_last
              person = Person.new
              person.first_name = "Austin"
              person.last_name = "Fonacier"

              assert_equal "Austin Fonacier", person.full_name
            end
          end
